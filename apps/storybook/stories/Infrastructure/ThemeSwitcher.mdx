# ThemeSwitcher

Theme toggle component with button, toggle, and dropdown variants.

## Overview

ThemeSwitcher allows users to change between light and dark themes:
- Multiple UI variants (button, toggle, dropdown)
- Persistent theme preference
- Smooth transitions
- System preference detection
- Accessible controls

<Canvas of={Stories.Default} />

## Use Cases

- **User preferences** - Allow theme customization
- **Accessibility** - Support user visual needs
- **App headers** - Common placement in navigation
- **Settings pages** - Part of appearance settings

## Accessibility

### Keyboard Navigation

- **Tab** - Focus theme switcher
- **Enter/Space** - Toggle theme (button/toggle)
- **Arrow keys** - Navigate dropdown options

### Screen Reader Support

- Clear labels ("Switch to dark theme")
- Current theme announced
- Role and state properly conveyed

## Best Practices

✅ **Do:**
- Persist theme preference (localStorage)
- Respect system preference by default
- Provide smooth transitions
- Place in accessible location (header)
- Include clear visual indicator

❌ **Don't:**
- Force theme without user control
- Use jarring transitions
- Hide theme switcher
- Forget to test both themes

## Examples

### Basic Usage

```tsx
import { ThemeSwitcher, ThemeProvider } from '@dynui-max/core';

function App() {
  return (
    <ThemeProvider defaultTheme="light">
      <header>
        <ThemeSwitcher variant="button" />
      </header>
      {/* App content */}
    </ThemeProvider>
  );
}
```

### With Custom Icons

```tsx
import { ThemeSwitcher } from '@dynui-max/core';

<ThemeSwitcher 
  variant="toggle"
  lightIcon="sun"
  darkIcon="moon"
/>
```
