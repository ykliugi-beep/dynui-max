# DynIcon

Icon system with size, color, and accessibility features.

## Overview

DynIcon provides a unified icon component:
- Design token-driven sizing and colors
- Accessible labels
- Customizable icon set
- Consistent styling across app

<Canvas of={Stories.Default} />

## Use Cases

- **Buttons** - Visual affordance for actions
- **Navigation** - Menu and breadcrumb icons
- **Status indicators** - Success, warning, error states
- **Form fields** - Input prefixes/suffixes

## Accessibility

- Decorative icons: `aria-hidden="true"`
- Meaningful icons: include accessible label
- Sufficient color contrast
- Don't rely solely on icons (pair with text)

## Best Practices

✅ **Do:**
- Use consistent icon sizes within context
- Pair icons with text labels when possible
- Choose recognizable, standard icons
- Use semantic colors (red=error, green=success)

❌ **Don't:**
- Use icons alone without labels for important actions
- Mix icon styles (outline vs filled)
- Use unclear or ambiguous icons
- Forget accessible labels

## Examples

### In Buttons

```tsx
import { DynButton, DynIcon } from '@dynui-max/core';

<DynButton>
  <DynIcon name="plus" size="sm" />
  Add Item
</DynButton>
```

### Icon Only (with label)

```tsx
import { DynIcon } from '@dynui-max/core';

<button aria-label="Close">
  <DynIcon name="close" aria-hidden="true" />
</button>
```
