# DynBadge

Status badges and labels with multiple color variants and sizes.

## Overview

DynBadge displays concise status indicators and labels:
- Multiple color variants for different states
- Size options for different contexts
- Pill and square border radius variants
- Icon support
- Accessible color contrast

<Canvas of={Stories.Default} />

## Use Cases

- **Status indicators** - Active, inactive, pending states
- **Labels and tags** - Categorization and filtering
- **Notifications** - Unread counts, alerts
- **Feature flags** - New, beta, deprecated markers

## Accessibility

- Sufficient color contrast (WCAG AA)
- Text alternatives for icon-only badges
- Semantic color usage (success=green, error=red)

## Best Practices

✅ **Do:**
- Use consistent colors for states across app
- Keep text concise (1-2 words)
- Choose appropriate size for context
- Use semantic colors (success, warning, error)

❌ **Don't:**
- Use too many badge colors
- Make badges interactive (use button instead)
- Use badges for lengthy text
- Rely solely on color to convey meaning
